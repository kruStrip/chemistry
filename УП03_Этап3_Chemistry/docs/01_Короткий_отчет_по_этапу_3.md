## 2. Выбранный вариант развертывания

**Вариант C — локальный demo-стенд через Docker.**

Причина: fullstack-проект (React + .NET + PostgreSQL + Redis + MinIO), уже имеет Docker-инфраструктуру после этапа 2; внешний VPS не обязателен для учебной демонстрации. Стенд поднимается production-командой `docker compose -f docker-compose.prod.yml up --build -d` и открывается в браузере вне IDE.

## 4. Команды развертывания

```bat
cd chemistry-backend\deploy
copy .env.demo.example .env.demo
cd ..
scripts\deploy.bat
scripts\check_deploy.bat
scripts\restart.bat
```

Эквивалент вручную:

```bat
docker compose -f deploy/docker-compose.demo.yml up --build -d
docker compose -f deploy/docker-compose.demo.yml ps
curl http://localhost:5000/health
```

## 5. Переменные окружения

| Файл | Назначение |
|------|------------|
| `deploy/.env.demo.example` | Demo-стенд: порты, mock-платежи, JWT_SECRET=change_me |
| `deploy/.env.production.example` | Шаблон для VPS/GHCR (без реальных секретов) |
| `chemistry-frontend/.env.demo.example` | VITE_* для сборки frontend-образа |

Рабочий файл `deploy/.env.demo` создаётся локально, **не коммитится**.

## 6. Проверка доступности

- Контейнеры `api`, `frontend`, `postgres`, `redis`, `minio` — **running**
- http://localhost:3000 — главная «Учим Химию» открывается в браузере
- `GET /health` → `{"status":"Healthy",...}`
- Swagger UI доступен, видны эндпоинты API
- `restart.bat` — down + up прошли успешно
- Логи: только WRN (DataProtection), критических ошибок нет

## 7. Проблемы и исправления

1. **GHCR-образы в `deploy/docker-compose.prod.yml`** — для локального стенда без registry добавлен `docker-compose.demo.yml` с `build:` из исходников.
2. **Production validator .NET** — для demo выбран `ASPNETCORE_ENVIRONMENT=Development` + mock-интеграции, иначе Production требует OzonPay/Captcha/Analytics.
3. **Конфликт портов с dev-стеком** — перед demo выполнен `docker compose down` dev-конфигурации.
