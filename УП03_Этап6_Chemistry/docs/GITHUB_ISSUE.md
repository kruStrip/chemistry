# GitHub Issue — шаблон инцидента (этап 6)

**Заголовок:** Ошибка входа: при неверном пароле показывается «Сессия истекла»

## Описание

При попытке входа с неверным паролем на странице `/login` пользователь видит сообщение «Сессия истекла или данные входа неверны», хотя сессия не истекала — введены неверные учётные данные.

## Шаги воспроизведения

1. Открыть http://localhost:3000/login
2. Ввести существующий email и заведомо неверный пароль
3. Нажать «Войти»
4. Наблюдать текст ошибки в форме

## Ожидаемое поведение

Понятное сообщение: «Неверный email или пароль».

## Фактическое поведение

Сообщение «Сессия истекла или данные входа неверны» — вводит в заблуждение.

## Окружение

- ОС: Windows 11
- Стек: .NET 8 API + React/Vite frontend
- Среда: local Docker demo (`deploy/docker-compose.demo.yml`)

## Диагностика

- Network: `POST /api/v1/auth/login` → 401, `title: Invalid credentials`
- Frontend: `getErrorMessage` не различал тип 401; `apiRequest` пытался refresh при 401 на login

**Связанные этапы:** BUG из эксплуатации после этапов 4–5 (роли/доступ проверены, UX входа не уточнялся).
