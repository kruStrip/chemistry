# Инструкция запуска проекта Chemistry

Дата аудита: 06.06.2026

## Требования

| Компонент | Версия |
|-----------|--------|
| .NET SDK | 8.0+ |
| Node.js | 20+ (проверено: 22.x) |
| Docker Desktop | 24+ (для полного стека) |
| Git | любая актуальная |

## Вариант A — Docker Compose (рекомендуется)

### 1. Клонировать репозитории

```powershell
git clone https://github.com/Studio-Kairos/chemistry-backend.git
git clone https://github.com/Studio-Kairos/chemistry-frontend.git
```

Репозитории должны лежать рядом: `chemistry-backend/` и `chemistry-frontend/`.

### 2. Настроить backend

```powershell
cd chemistry-backend/deploy
Copy-Item .env.example .env
# Отредактируйте .env: POSTGRES_PASSWORD, REDIS_PASSWORD, MINIO_*, JWT_SECRET (мин. 32 символа)
docker compose up -d
```

Первый запуск может занять 5–15 минут (сборка образа API).

### 3. Проверить сервисы

| Сервис | URL |
|--------|-----|
| API | http://localhost:5000 |
| Swagger | http://localhost:5000/swagger |
| Health | http://localhost:5000/health |
| Frontend (контейнер) | http://localhost:3000 |
| Grafana | http://localhost:3001 |
| MinIO Console | http://localhost:9001 |

### 4. Запустить frontend отдельно (разработка)

Если контейнер `frontend` не используется:

```powershell
cd chemistry-frontend
npm ci
Copy-Item .env.example .env
npm run dev
```

В `.env` для dev-режима:
```
VITE_API_BASE_URL=http://localhost:3000
VITE_API_PROXY_TARGET=http://127.0.0.1:5000
```

## Вариант B — dotnet run + Docker-зависимости

```powershell
cd chemistry-backend/deploy
docker compose up -d postgres redis minio minio-init loki

cd chemistry-backend
$env:ASPNETCORE_ENVIRONMENT="Development"
$env:ConnectionStrings__Postgres="Host=localhost;Port=5432;Database=chemistry_db;Username=chemistry_user;Password=<из .env>"
$env:JWT_SECRET="<из .env>"
$env:Redis__ConnectionString="localhost:6379,password=<REDIS_PASSWORD>,abortConnect=false"
dotnet run --project src/Chemistry.Api/Chemistry.Api.csproj --urls http://localhost:5000
```

## Миграции БД

```powershell
dotnet run --project src/Chemistry.Api/Chemistry.Api.csproj -- --migrate
```

При `Database__ApplyMigrationsOnStartup=true` миграции применяются автоматически при старте (кроме Production).

## Тесты

```powershell
# Backend unit-тесты
dotnet test tests/Chemistry.UnitTests/Chemistry.UnitTests.csproj

# Frontend unit-тесты
cd chemistry-frontend
npm run test:run
```

## Протокол запуска (чеклист)

| № | Действие | Команда | Ожидаемый результат |
|---|----------|---------|----------------------|
| 1 | Клонировать backend | `git clone https://github.com/Studio-Kairos/chemistry-backend.git` | Папка `chemistry-backend` |
| 2 | Клонировать frontend | `git clone https://github.com/Studio-Kairos/chemistry-frontend.git` | Папка `chemistry-frontend` |
| 3 | Создать .env backend | `Copy-Item .env.example .env` | Файл `.env` в `deploy/` |
| 4 | Поднять стек | `docker compose up -d` | Контейнеры Running |
| 5 | Проверить API | `curl http://localhost:5000/health` | `{"status":"Healthy",...}` |
| 6 | Установить frontend deps | `npm ci` | Без критических ошибок |
| 7 | Запустить dev-сервер | `npm run dev` | `http://localhost:3000` |
| 8 | Открыть Swagger | http://localhost:5000/swagger | Документация API |

## Известные ограничения

- На чистой локальной БД каталог курсов пуст (нет автосида) — курсы создаются через админку.
- Маршрут `/register` на фронте не подключён в `router.tsx` (страница есть, ссылка с login битая).
- `deploy/.env.test` отсутствует — `docker-compose.test.yml` не запустится без ручного создания.
